# WarehousePG 7.x — Tiered Partitioned Table with PXF + MinIO
## Hands-On Lab Guide

**Reference:** [Exchanging a Leaf Child Partition with an External Table](https://warehouse-pg.io/docs/6x/admin_guide/ddl/ddl-partition/#topic_yhz_gpn_qs)  
**Catalog migration:** [Migrating Partition Maintenance Scripts to WarehousePG 7](https://www.enterprisedb.com/docs/warehousepg/latest/install_guide/migrate-classic-partitioning.mdx)

> **Key change from 6.x → 7.x**  
> `pg_partitions`, `pg_partition`, `pg_partition_rule`, and related views are
> **removed** in WarehousePG 7. Use `pg_partition_tree()`, `pg_class`, and
> `pg_get_expr(relpartbound, oid)` instead.

---

## Lab Architecture

```
┌──────────────────────────────────────────────────────────────────────┐
│  events  (parent partitioned table — single query entry point)       │
│                                                                      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────────────┐   │
│  │ p202506      │  │ p202505      │  │ p202503_ext              │   │
│  │ HEAP (hold)  │  │ HEAP (warm)  │  │ PXF External  ──► MinIO  │   │
│  │ NVMe, R/W    │  │ HDD, R-heavy │  │ Parquet, read-only       │   │
│  └──────────────┘  └──────────────┘  └──────────────────────────┘   │
└──────────────────────────────────────────────────────────────────────┘

Exchange flow (warm → cold):
  ① CREATE WRITABLE EXTERNAL TABLE   (MinIO export sink)
  ② INSERT INTO writable ext          (export data as Parquet)
  ③ CREATE EXTERNAL TABLE             (MinIO readable, same path)
  ④ ALTER TABLE … EXCHANGE PARTITION … WITHOUT VALIDATION
  ⑤ DROP writable ext + old heap shard
  ⑥ RENAME PARTITION to _ext
```

## Prerequisites

| Item | Requirement |
|------|-------------|
| WarehousePG | **7.x** cluster (coordinator + ≥ 1 segment host) |
| PXF | Installed and running on every segment host |
| MinIO | Accessible from all segment hosts |
| `mc` CLI | MinIO client installed on the coordinator |

> **Constraint (unchanged from 6.x):** `EXCHANGE PARTITION` with an external
> table fails if the parent table has a `NOT NULL` or `CHECK` constraint.

## Part 1 — Create the Parent Partitioned Table

```sql
-- Enable PXF in this database (run once)
CREATE EXTENSION IF NOT EXISTS pxf;

GRANT SELECT ON PROTOCOL pxf TO gpadmin;
GRANT INSERT ON PROTOCOL pxf TO gpadmin;

-- Parent partitioned table
-- No NOT NULL / CHECK constraints → required for EXCHANGE PARTITION with ext table
CREATE TABLE events (
    event_id    BIGINT,
    tenant_id   INT,
    event_type  VARCHAR(64),
    payload     TEXT,
    region      TEXT,
    amount      NUMERIC(12, 2),
    created_at  TIMESTAMP
)
DISTRIBUTED BY (tenant_id)
PARTITION BY RANGE (created_at)
(
    PARTITION p202506
        START (TIMESTAMP '2025-06-01') INCLUSIVE
        END   (TIMESTAMP '2025-07-01') EXCLUSIVE,
    PARTITION p202505
        START (TIMESTAMP '2025-05-01') INCLUSIVE
        END   (TIMESTAMP '2025-06-01') EXCLUSIVE,
    PARTITION p202504
        START (TIMESTAMP '2025-04-01') INCLUSIVE
        END   (TIMESTAMP '2025-05-01') EXCLUSIVE,
    PARTITION p202503
        START (TIMESTAMP '2025-03-01') INCLUSIVE
        END   (TIMESTAMP '2025-04-01') EXCLUSIVE,
    PARTITION p202502
        START (TIMESTAMP '2025-02-01') INCLUSIVE
        END   (TIMESTAMP '2025-03-01') EXCLUSIVE,
    PARTITION p202501
        START (TIMESTAMP '2025-01-01') INCLUSIVE
        END   (TIMESTAMP '2025-02-01') EXCLUSIVE
);
```

### Verify partition structure (WarehousePG 7 way)

In WarehousePG 7, `pg_partitions` is removed. Use `pg_partition_tree()` and
`pg_get_expr()` instead:

```sql
-- List all partitions with their boundary and storage type
SELECT
    pt.relid::regclass                          AS partition_table,
    pt.level,
    pt.isleaf,
    pg_get_expr(c.relpartbound, c.oid)          AS partition_bound,
    CASE c.relkind
        WHEN 'r' THEN 'heap (local)'
        WHEN 'f' THEN 'external (PXF)'
    END                                         AS storage_type
FROM   pg_partition_tree('events') pt
JOIN   pg_class c ON c.oid = pt.relid
WHERE  pt.isleaf = true
ORDER  BY partition_bound;
```

Expected output:

```
      partition_table       │ level │ isleaf │              partition_bound               │ storage_type
────────────────────────────┼───────┼────────┼────────────────────────────────────────────┼──────────────
 events_1_prt_p202501       │   1   │ t      │ FOR VALUES FROM ('2025-01-01') TO ('2025-02-01') │ heap (local)
 events_1_prt_p202502       │   1   │ t      │ FOR VALUES FROM ('2025-02-01') TO ('2025-03-01') │ heap (local)
 events_1_prt_p202503       │   1   │ t      │ FOR VALUES FROM ('2025-03-01') TO ('2025-04-01') │ heap (local)
 events_1_prt_p202504       │   1   │ t      │ FOR VALUES FROM ('2025-04-01') TO ('2025-05-01') │ heap (local)
 events_1_prt_p202505       │   1   │ t      │ FOR VALUES FROM ('2025-05-01') TO ('2025-06-01') │ heap (local)
```

## Part 3 — Exchange One Partition to Cold (Manual Walkthrough)

We archive **January 2025** (`events_1_prt_p202501`) to MinIO step by step.

> In WarehousePG 7, `EXCHANGE PARTITION` is internally converted to
> `DETACH PARTITION` + `ATTACH PARTITION`. The classic syntax still works
> but the underlying mechanism has changed.

### Step 1 — Create a writable PXF external table (export sink)

```sql
CREATE WRITABLE EXTERNAL TABLE events_cold_write_202501
    (LIKE events_1_prt_p202501)
LOCATION (
    'pxf://warehouse-lab/events/year=2025/month=01?PROFILE=s3:parquet&SERVER=miniocfg'
)
FORMAT 'CUSTOM' (FORMATTER = 'pxfwritable_export')
DISTRIBUTED BY (tenant_id);
```

### Step 2 — Export data to MinIO as Parquet

```sql
INSERT INTO events_cold_write_202501
SELECT * FROM events_1_prt_p202501;
```

Verify on MinIO:

```bash
mc ls local/warehouse-lab/events/year=2025/month=01/
# Expect one .parquet file per segment
```

### Step 3 — Create a readable PXF external table (same path)

```sql
CREATE EXTERNAL TABLE events_cold_read_202501
    (LIKE events_1_prt_p202501)
LOCATION (
    'pxf://warehouse-lab/events/year=2025/month=01?PROFILE=s3:parquet&SERVER=miniocfg'
)
FORMAT 'CUSTOM' (FORMATTER = 'pxfwritable_import');
```

Spot-check:

```sql
SELECT COUNT(*)   FROM events_cold_read_202501;
SELECT event_type, SUM(amount) FROM events_cold_read_202501 GROUP BY 1;
```

### Step 4 — Exchange the heap partition with the readable external table

```sql
-- In WarehousePG 7 the partition-to-exchange must have the same
-- constraint as the parent table. WITHOUT VALIDATION skips the data check;
-- safe here because we exported exactly the rows that belong to this range.
-- 단일 레벨 파티션: ALTER PARTITION 없이 바로 EXCHANGE PARTITION
ALTER TABLE events
    EXCHANGE PARTITION p202501
    WITH TABLE events_cold_read_202501;
```

After the swap:
- `events_1_prt_p202501` → now backed by MinIO via PXF
- `events_cold_read_202501` → now holds the old heap shard (renamed by the swap)

### Step 5 — Clean up

```sql
DROP EXTERNAL TABLE IF EXISTS events_cold_write_202501;
DROP TABLE        IF EXISTS events_cold_read_202501;   -- reclaims local disk
```

### Step 6 — Rename the partition (optional, for clarity)

```sql
ALTER TABLE events RENAME PARTITION p202501 TO p202501_ext;
```

### Step 7 — Verify with the 7.x catalog queries

```sql
-- Check storage type per partition (relkind 'r'=heap, 'f'=external/foreign)
SELECT
    pt.relid::regclass                          AS partition_table,
    CASE c.relkind
        WHEN 'r' THEN 'heap  (hold / warm)'
        WHEN 'f' THEN 'external (cold → MinIO)'
    END                                         AS storage_tier,
    pg_get_expr(c.relpartbound, c.oid)          AS partition_bound
FROM   pg_partition_tree('events') pt
JOIN   pg_class c ON c.oid = pt.relid
WHERE  pt.isleaf = true
ORDER  BY partition_bound;

-- The parent table still returns all 10000 rows transparently
SELECT COUNT(*) FROM events;

-- Confirm January rows are served from MinIO
SELECT COUNT(*)
FROM   events
WHERE  created_at >= '2025-01-01'
AND    created_at <  '2025-02-01';

-- EXPLAIN: cold partition shows Foreign Scan, hot partitions show Seq Scan
EXPLAIN SELECT * FROM events WHERE created_at = '2025-01-15 10:00:00';
EXPLAIN SELECT * FROM events WHERE created_at = '2025-06-15 10:00:00';
```

## Part 4 — Automate with a Stored Procedure
```
cat > /tmp/archive_proc.sql << 'HEREDOC'
CREATE OR REPLACE PROCEDURE archive_partition_to_minio(
    p_year  INT,
    p_month INT
)
LANGUAGE plpgsql AS $proc$
DECLARE
    v_month_pad   TEXT;
    v_part_name   TEXT;
    v_heap_table  TEXT;
    v_s3_path     TEXT;
    v_write_ext   TEXT;
    v_read_ext    TEXT;
    v_heap_exists INT;
    v_row_count   BIGINT;
BEGIN
    -- Build zero-padded month once, reuse in all variable assignments
    v_month_pad  := lpad(p_month::TEXT, 2, '0');

    v_part_name  := format('p%s%s',  p_year, v_month_pad);
    v_heap_table := format('events_1_prt_%s', v_part_name);
    v_s3_path    := format('warehouse-lab/events/year=%s/month=%s',
                            p_year, v_month_pad);
    v_write_ext  := format('events_cold_write_%s_%s', p_year, v_month_pad);
    v_read_ext   := format('events_cold_read_%s_%s', p_year, v_month_pad);

    RAISE NOTICE '[%] Starting archive: %', clock_timestamp(), v_part_name;
    RAISE NOTICE '[%] S3 path: %',          clock_timestamp(), v_s3_path;

    -- Guard: skip if already external or does not exist
    SELECT COUNT(*) INTO v_heap_exists
    FROM   pg_class
    WHERE  relname = v_heap_table
    AND    relkind = 'r';

    IF v_heap_exists = 0 THEN
        RAISE NOTICE 'Partition % is not a heap table, skipping.', v_part_name;
        RETURN;
    END IF;

    -- Row count from heap BEFORE export (writable ext cannot be SELECTed)
    EXECUTE format('SELECT COUNT(*) FROM %I', v_heap_table)
        INTO v_row_count;
    RAISE NOTICE '[%] Rows to export: %', clock_timestamp(), v_row_count;

    -- STEP 1: writable external table -> MinIO
    EXECUTE format(
        'CREATE WRITABLE EXTERNAL TABLE %I (LIKE %I)
         LOCATION (''pxf://%s?PROFILE=s3:parquet&SERVER=miniocfg'')
         FORMAT ''CUSTOM'' (FORMATTER=''pxfwritable_export'')
         DISTRIBUTED BY (tenant_id)',
        v_write_ext, v_heap_table, v_s3_path
    );

    -- STEP 2: export heap data to MinIO as Parquet
    EXECUTE format(
        'INSERT INTO %I SELECT * FROM %I',
        v_write_ext, v_heap_table
    );
    RAISE NOTICE '[%] Exported % rows to %',
                 clock_timestamp(), v_row_count, v_s3_path;

    -- STEP 3: readable external table over the same MinIO path
    EXECUTE format(
        'CREATE EXTERNAL TABLE %I (LIKE %I)
         LOCATION (''pxf://%s?PROFILE=s3:parquet&SERVER=miniocfg'')
         FORMAT ''CUSTOM'' (FORMATTER=''pxfwritable_import'')',
        v_read_ext, v_heap_table, v_s3_path
    );

    -- STEP 4: exchange heap partition with the readable external table
    EXECUTE format(
        'ALTER TABLE events EXCHANGE PARTITION %I WITH TABLE %I',
        v_part_name, v_read_ext
    );
    RAISE NOTICE '[%] Partition % exchanged with PXF external table',
                 clock_timestamp(), v_part_name;

    -- STEP 5: drop temp tables and reclaim local disk
    EXECUTE format('DROP EXTERNAL TABLE IF EXISTS %I', v_write_ext);
    EXECUTE format('DROP TABLE        IF EXISTS %I', v_read_ext);

    -- STEP 6: rename partition to mark cold status
    EXECUTE format(
        'ALTER TABLE events RENAME PARTITION %I TO %I',
        v_part_name, v_part_name || '_ext'
    );

    RAISE NOTICE '[%] Done. %_ext is now served from MinIO.',
                 clock_timestamp(), v_part_name;

EXCEPTION WHEN OTHERS THEN
    EXECUTE format('DROP EXTERNAL TABLE IF EXISTS %I', v_write_ext);
    EXECUTE format('DROP EXTERNAL TABLE IF EXISTS %I', v_read_ext);
    RAISE;
END;
$proc$;
HEREDOC
```

```
psql -U gpadmin -d your_database -f /tmp/archive_proc.sql
```
