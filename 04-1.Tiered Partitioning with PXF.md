# WarehousePG — Tiered Partitioning with PXF (No View Required)

Hot and warm partitions stay as local heap tables. When a partition ages past the
cold threshold, it is replaced in-place with a PXF external table pointing to
S3 or MinIO. The parent table remains the single query target — no view, no
application changes.

---

## How It Works

The key mechanism is `ALTER TABLE … EXCHANGE PARTITION … WITH TABLE`. Swapping a
readable PXF external table into a partition slot makes that partition
transparently backed by object storage. Any `SELECT` against the parent table
automatically routes to the right backend per partition.

```
events (parent — PARTITION BY RANGE)
├── p202506   HEAP        ← hold  (NVMe, read/write)
├── p202505   HEAP        ← warm  (HDD, read-heavy)
├── p202504   HEAP        ← warm
├── p202503   PXF ext     ← cold  (MinIO / S3 Parquet)
├── p202502   PXF ext     ← cold
└── p202501   PXF ext     ← cold
```

---

## Prerequisites

- WarehousePG 7.x with PXF installed and running on every segment host
- PXF server configuration synced via `pxf cluster sync`
- Parent table must **not** have `NOT NULL` or `CHECK` constraints
  (these block `EXCHANGE PARTITION` with an external table)

---

## Step 1 — Create the Parent Partitioned Table

```sql
-- Avoid NOT NULL / CHECK constraints — they block external table exchange
CREATE TABLE events (
    event_id    BIGINT,
    tenant_id   INT,
    event_type  VARCHAR(64),
    payload     JSONB,
    created_at  TIMESTAMPTZ DEFAULT now()
)
DISTRIBUTED BY (tenant_id)
PARTITION BY RANGE (created_at)
(
    PARTITION p202506 START ('2025-06-01') INCLUSIVE END ('2025-07-01') EXCLUSIVE,
    PARTITION p202505 START ('2025-05-01') INCLUSIVE END ('2025-06-01') EXCLUSIVE,
    PARTITION p202504 START ('2025-04-01') INCLUSIVE END ('2025-05-01') EXCLUSIVE,
    PARTITION p202503 START ('2025-03-01') INCLUSIVE END ('2025-04-01') EXCLUSIVE,
    PARTITION p202502 START ('2025-02-01') INCLUSIVE END ('2025-03-01') EXCLUSIVE,
    PARTITION p202501 START ('2025-01-01') INCLUSIVE END ('2025-02-01') EXCLUSIVE
);
```

> The internal heap table for each partition is auto-named `events_1_prt_<partition_name>`.

  </property>

    <name>fs.s3a.path.style.access</name>

`fs.s3a.secret.key`, and the regional endpoint (e.g.

## Step 4 — Cold Archival Procedure (Heap → External Table Exchange)

This single stored procedure encapsulates all four steps of the exchange:

```sql
CREATE OR REPLACE PROCEDURE exchange_partition_to_cold(
    p_year  INT,
    p_month INT
)
LANGUAGE plpgsql AS $$
DECLARE
    -- Partition name used in ALTER TABLE clauses
    v_part_name  TEXT := format('p%s%s', p_year, lpad(p_month::TEXT, 2, '0'));
    -- Internal heap table name created by WarehousePG
    v_heap_table TEXT := format('events_1_prt_%s', v_part_name);
    -- Object store path (Hive-style layout)
    v_s3_path    TEXT := format(
                             'warehouse-bucket/events/year=%s/month=%s',
                             p_year, lpad(p_month::TEXT, 2, '0')
                         );
    v_write_ext  TEXT := format('events_cold_write_%s_%s', p_year, p_month);
    v_read_ext   TEXT := format('events_cold_read_%s_%s',  p_year, p_month);
BEGIN
    RAISE NOTICE 'Starting cold archival for %-%', p_year, p_month;

    -- Step 1: Create a writable external table to export data to MinIO
    EXECUTE format($$
        CREATE WRITABLE EXTERNAL TABLE %I (LIKE %I)
        LOCATION ('pxf://%s?PROFILE=s3:parquet&SERVER=miniocfg')
        FORMAT 'CUSTOM' (FORMATTER='pxfwritable_export')
        DISTRIBUTED BY (tenant_id)
    $$, v_write_ext, v_heap_table, v_s3_path);

    -- Step 2: Export heap data to MinIO as Parquet
    EXECUTE format('INSERT INTO %I SELECT * FROM %I', v_write_ext, v_heap_table);
    RAISE NOTICE 'Exported to MinIO: %', v_s3_path;

    -- Step 3: Create a readable external table over the same path
    EXECUTE format($$
        CREATE EXTERNAL TABLE %I (LIKE %I)
        LOCATION ('pxf://%s?PROFILE=s3:parquet&SERVER=miniocfg')
        FORMAT 'CUSTOM' (FORMATTER='pxfwritable_import')
    $$, v_read_ext, v_heap_table, v_s3_path);

    -- Step 4: Swap the readable external table into the partition slot.
    --         WITHOUT VALIDATION is safe here because we just exported
    --         exactly the data that belongs to this partition's range.
    EXECUTE format($$
        ALTER TABLE events ALTER PARTITION %I
            EXCHANGE PARTITION %I
            WITH TABLE %I WITHOUT VALIDATION
    $$, v_part_name, v_part_name, v_read_ext);

    RAISE NOTICE 'Partition % swapped with PXF external table', v_part_name;

    -- Cleanup: drop the writable ext table; the old heap shard is now
    -- named v_read_ext after the swap — drop it to reclaim local disk
    EXECUTE format('DROP EXTERNAL TABLE IF EXISTS %I', v_write_ext);
    EXECUTE format('DROP TABLE IF EXISTS %I', v_read_ext);

    -- Optional: rename partition to make the _ext status visible
    EXECUTE format(
        'ALTER TABLE events RENAME PARTITION %I TO %I',
        v_part_name, v_part_name || '_ext'
    );

    RAISE NOTICE 'Done: partition %_ext is now backed by MinIO', v_part_name;
END;
$$;
```

## Step 5 — Manual Execution

```sql
-- Archive March 2025 to cold storage
CALL exchange_partition_to_cold(2025, 3);

-- Archive January and February as well
CALL exchange_partition_to_cold(2025, 2);
CALL exchange_partition_to_cold(2025, 1);
```

---

## Step 6 — Automate with pg_cron (90-day rolling policy)

```sql
CREATE EXTENSION IF NOT EXISTS pg_cron;

-- Run on the 1st of every month at 02:00
-- Archives the partition that is exactly 3 months old
SELECT cron.schedule(
    'auto-cold-archive',
    '0 2 1 * *',
    $$
    CALL exchange_partition_to_cold(
        extract(year  FROM (CURRENT_DATE - INTERVAL '90 days'))::INT,
        extract(month FROM (CURRENT_DATE - INTERVAL '90 days'))::INT
    )
    $$
);
```

## Step 8 — Querying All Tiers Through the Parent Table

After the exchange, the parent table routes each query to the correct backend
automatically. No view or application change required.

```sql
-- Single table scan spans hold (heap) + warm (heap) + cold (PXF / MinIO)
SELECT
    date_trunc('month', created_at) AS month,
    event_type,
    count(*)
FROM events
WHERE tenant_id = 42
  AND created_at >= '2025-01-01'
GROUP BY 1, 2
ORDER BY 1 DESC;
```

To verify which partitions are heap and which are external:

```sql
-- Check access method per partition
SELECT
    c.relname        AS partition_table,
    CASE WHEN c.relkind = 'f' THEN 'external/foreign'
         ELSE am.amname
    END              AS access_method
FROM pg_class c
LEFT JOIN pg_am am ON c.relam = am.oid
WHERE c.relname LIKE 'events_1_prt_%'
  AND c.relkind IN ('r', 'f')
ORDER BY c.relname;

-- Use EXPLAIN to confirm PXF routing for a cold partition
EXPLAIN SELECT * FROM events WHERE created_at = '2025-03-15';
-- A "Foreign Scan" node in the plan confirms PXF is handling that partition
```

## PXF Profile Reference

| Format  | Profile (MinIO / S3)  | Read | Write |
|---------|-----------------------|:----:|:-----:|
| Parquet | `s3:parquet`          |  ✓   |   ✓   |
| ORC     | `s3:orc`              |  ✓   |   ✓   |
| CSV     | `s3:text` / `s3:csv`  |  ✓   |   ✓   |
| JSON    | `s3:json`             |  ✓   |   ✓   |
| Avro    | `s3:avro`             |  ✓   |   ✓   |

## Known Limitations

| Limitation | Detail |
|------------|--------|
| `NOT NULL` / `CHECK` constraints | Partitions with these constraints on the parent table cannot be exchanged with an external table. Remove them before using this pattern. |
| Cold partitions are read-only | Once exchanged, the partition is backed by an external table and does not support `INSERT`, `UPDATE`, or `DELETE`. |
| `WITHOUT VALIDATION` risk | WarehousePG skips constraint checking on exchange. Ensure the exported data strictly matches the partition's date range to avoid incorrect query results. |
| `gpbackup` | Data in external table partitions is **not** backed up by `gpbackup`. Back up the object store separately. |
| GPORCA optimizer | GPORCA does not support multi-level partitioned tables. Queries against multi-level hierarchies fall back to the Postgres-based planner automatically. |

## Architecture Summary

```
┌─────────────────────────────────────────────────────────────────┐
│  events (parent partitioned table)                              │
│                                                                 │
│  p202506  ──→  HEAP  (NVMe)   ← hold:  writes land here        │
│  p202505  ──→  HEAP  (HDD)    ← warm:  recent reads            │
│  p202504  ──→  HEAP  (HDD)    ← warm                           │
│  p202503_ext ──→  PXF ext  ──→  MinIO/S3  ← cold: Parquet      │
│  p202502_ext ──→  PXF ext  ──→  MinIO/S3  ← cold               │
│  p202501_ext ──→  PXF ext  ──→  MinIO/S3  ← cold               │
└─────────────────────────────────────────────────────────────────┘

SELECT * FROM events WHERE ...   ← single entry point, zero app changes
```
