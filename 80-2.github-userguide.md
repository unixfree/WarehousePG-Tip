# Git 작업 흐름 가이드: Clone → Branch → Commit → Push → Merge

## 1. 저장소 클론

```bash
git clone https://github.com/unixfree/WarehousePG-Tip.git
cd WarehousePG-Tip
```

클론하면 자동으로 기본 브랜치(보통 `main`)가 체크아웃됩니다.

## 2. 새 브랜치 생성 및 이동

```bash
# 브랜치 생성과 동시에 이동 (권장)
git checkout -b feature/my-work

# 또는 최신 Git 방식
git switch -c feature/my-work
```

`-b`(또는 `-c`)는 브랜치를 새로 만들면서 그 브랜치로 전환합니다. 현재 브랜치 확인은 `git branch` 또는 `git status`로 할 수 있습니다.

## 3. 파일 수정 후 커밋

```bash
# 파일을 수정/추가한 뒤
git add .                    # 모든 변경사항 스테이징
# 또는 특정 파일만: git add 파일명

git commit -m "기능 추가: 설명 메시지"
```

여러 번 작업하며 커밋을 반복할 수 있습니다. 커밋 전에 `git status`로 무엇이 스테이징됐는지, `git diff`로 변경 내용을 확인하는 습관이 좋습니다.

## 4. 원격 저장소로 푸시

```bash
git push -u origin feature/my-work
```

`-u`(`--set-upstream`)는 로컬 브랜치와 원격 브랜치를 연결합니다. 한 번 설정하면 이후엔 `git push`만 입력해도 됩니다.

> 푸시하려면 저장소에 대한 쓰기 권한이 필요합니다. 권한이 없는 경우 먼저 fork한 뒤 본인 저장소로 푸시해야 합니다.

## 5. 머지 (Merge)

머지는 두 가지 방식이 있습니다.

### 방식 A — GitHub Pull Request (협업 시 권장)

푸시 후 GitHub 저장소 페이지에 가면 "Compare & pull request" 버튼이 나타납니다. PR을 생성하고 리뷰를 거친 뒤 "Merge pull request" 버튼으로 머지합니다. 협업 프로젝트에서는 이 방식이 표준입니다.

### 방식 B — 로컬에서 직접 머지

```bash
git checkout main            # 머지 대상 브랜치로 이동
git pull origin main         # 최신 상태로 업데이트
git merge feature/my-work    # 작업 브랜치를 병합
git push origin main         # 병합 결과를 원격에 반영
```

## 6. 정리 (선택)

머지가 끝나면 작업 브랜치를 삭제할 수 있습니다.

```bash
git branch -d feature/my-work              # 로컬 브랜치 삭제
git push origin --delete feature/my-work   # 원격 브랜치 삭제
```

## 충돌(Conflict)이 발생하면

머지 시 같은 부분을 다르게 수정했다면 충돌이 납니다. 충돌 파일을 열어 `<<<<<<<`, `=======`, `>>>>>>>` 표시 부분을 직접 수정한 뒤:

```bash
git add 충돌해결한파일
git commit              # 머지 커밋 완성
```

## 전체 흐름 요약

```
clone → branch 생성 → 수정 → add → commit → push → (PR 또는 merge) → push
```

# Git 명령어 모음 (용도별 정리)

## 1. 초기 설정

```bash
git config --global user.name "홍길동"           # 사용자 이름 설정
git config --global user.email "hong@example.com" # 이메일 설정
git config --list                                 # 현재 설정 확인
git config --global init.defaultBranch main       # 기본 브랜치 이름 지정
```

## 2. 저장소 생성 / 가져오기

```bash
git init                                          # 현재 폴더를 Git 저장소로 초기화
git clone https://github.com/user/repo.git        # 원격 저장소 복제
git clone https://github.com/user/repo.git 폴더명  # 다른 폴더명으로 복제
git clone -b 브랜치명 URL                          # 특정 브랜치만 복제
```

## 3. 상태 확인 및 변경 내용 보기

```bash
git status                  # 현재 작업 상태 확인
git status -s               # 짧은 형식으로 확인
git diff                    # 스테이징 안 된 변경 내용
git diff --staged           # 스테이징 된 변경 내용 (= --cached)
git diff main feature       # 두 브랜치 간 차이
git show 커밋해시            # 특정 커밋의 상세 내용
```

## 4. 스테이징 및 커밋

```bash
git add 파일명               # 특정 파일 스테이징
git add .                    # 모든 변경 스테이징
git add -u                   # 추적 중인 파일의 수정/삭제만 스테이징
git add -p                   # 변경 부분을 조각 단위로 선택해 스테이징

git commit -m "메시지"        # 커밋
git commit -am "메시지"       # 추적 중인 파일 add + commit 동시에
git commit --amend           # 마지막 커밋 수정 (메시지/내용)
git restore --staged 파일명   # 스테이징 취소 (작업 내용은 유지)
```

## 5. 브랜치 관리

```bash
git branch                   # 로컬 브랜치 목록
git branch -a                # 원격 포함 모든 브랜치
git branch 브랜치명           # 브랜치 생성
git checkout 브랜치명         # 브랜치 전환
git checkout -b 브랜치명      # 생성 + 전환
git switch 브랜치명           # 브랜치 전환 (최신 방식)
git switch -c 브랜치명        # 생성 + 전환 (최신 방식)
git branch -d 브랜치명        # 브랜치 삭제 (병합된 것만)
git branch -D 브랜치명        # 브랜치 강제 삭제
git branch -m 새이름          # 현재 브랜치 이름 변경
```

## 6. 병합 및 리베이스

```bash
git merge 브랜치명            # 현재 브랜치에 지정 브랜치 병합
git merge --no-ff 브랜치명    # 병합 커밋을 항상 생성
git merge --abort            # 병합 충돌 시 병합 취소
git rebase main              # 현재 브랜치를 main 위로 재배치
git rebase --abort           # 리베이스 취소
git rebase -i HEAD~3         # 최근 3개 커밋 대화형 정리(스쿼시 등)
```

## 7. 원격 저장소 연동

```bash
git remote -v                       # 원격 저장소 목록 확인
git remote add origin URL           # 원격 저장소 추가
git remote remove origin            # 원격 저장소 제거
git fetch origin                    # 원격 변경사항 가져오기 (병합 X)
git pull origin main                # 가져오기 + 병합
git pull --rebase origin main       # 가져오기 + 리베이스
git push -u origin 브랜치명          # 첫 푸시 (업스트림 설정)
git push                            # 이후 푸시
git push origin --delete 브랜치명    # 원격 브랜치 삭제
```

## 8. 변경 취소 / 되돌리기

```bash
git restore 파일명            # 작업 디렉토리 수정 취소
git checkout -- 파일명        # (구버전 방식) 수정 취소
git reset HEAD~1             # 마지막 커밋 취소 (변경은 유지, unstaged)
git reset --soft HEAD~1      # 마지막 커밋 취소 (변경은 staged 유지)
git reset --hard HEAD~1      # 마지막 커밋 + 변경 내용 완전 삭제 ⚠️
git revert 커밋해시           # 특정 커밋을 되돌리는 새 커밋 생성 (안전)
git clean -fd               # 추적 안 되는 파일/폴더 삭제 ⚠️
```

> ⚠️ `reset --hard`와 `clean`은 복구가 어려우니 주의해서 사용하세요.

## 9. 임시 저장 (Stash)

작업 중인 내용을 커밋하지 않고 잠시 치워둘 때 사용합니다.

```bash
git stash                    # 변경 내용 임시 저장
git stash -u                 # 추적 안 되는 파일도 포함
git stash list               # 저장된 stash 목록
git stash pop                # 가장 최근 stash 복원 후 삭제
git stash apply              # 복원하되 stash는 남겨둠
git stash drop               # 특정 stash 삭제
```

## 10. 히스토리 조회

```bash
git log                      # 커밋 히스토리
git log --oneline            # 한 줄로 간략히
git log --oneline --graph    # 그래프 형태로
git log -p                   # 변경 내용까지 표시
git log --author="홍길동"     # 특정 작성자 커밋만
git log 파일명               # 특정 파일의 히스토리
git blame 파일명             # 각 줄을 누가 마지막으로 수정했는지
```

## 11. 태그 (버전 표시)

```bash
git tag                      # 태그 목록
git tag v1.0.0               # 가벼운 태그 생성
git tag -a v1.0.0 -m "릴리스" # 주석 태그 생성
git push origin v1.0.0       # 태그 푸시
git push origin --tags       # 모든 태그 푸시
```

## 자주 쓰는 실전 시나리오

### 실수로 커밋한 메시지 수정

```bash
git commit --amend -m "올바른 메시지"
```

### 잘못된 브랜치에서 작업한 경우 (stash 활용)

```bash
git stash                    # 변경 임시 저장
git switch 올바른브랜치       # 브랜치 이동
git stash pop                # 변경 복원
```

### 원격의 최신 변경을 안전하게 반영

```bash
git fetch origin             # 먼저 가져와서 확인
git log origin/main          # 원격 상태 확인
git merge origin/main        # 문제 없으면 병합
```
